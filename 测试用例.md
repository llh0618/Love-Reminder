## ADB 自动化测试局限性

以下操作 ADB 无法可靠执行，需手动验证：
1. 中文文本输入（`adb input text` 仅支持 ASCII）
2. 日期选择器清空和输入（inputType="date" 触发特殊键盘）
3. 系统分享菜单操作（FileProvider URI）
4. 长按删除操作
5. 权限弹窗点击

## 发现的 Bug 及修复

### Bug #1: DURATION + DTEND 冲突 【已修复】
- **现象**：保存纪念日时崩溃 `Cannot have both DTEND and DURATION in an event`
- **根因**：`CalendarHelper.kt:93` 同时设置了 DTEND 和 DURATION
- **修复**：删除 DURATION 行，提前提醒由 Alarm 机制处理
